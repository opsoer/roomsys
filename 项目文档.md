# 项目文档 — 租房财务管理系统

## 一、项目概述

**项目名称**：租房财务管理系统（多租户 SaaS 版）  
**项目定位**：面向深圳本地合伙制公寓租赁业务的多房东 SaaS 财务管理平台  
**当前状态**：已完成多租户改造，支持平台超级管理员管理多个楼栋，每个楼栋独立运营

---

## 二、部署指南

### 环境要求

| 组件 | 版本要求 |
|------|---------|
| MySQL | 5.7+ |
| Go | 1.25+ |
| Node.js | 18+ |

### 2.1 数据库准备

创建 MySQL 数据库：

```sql
CREATE DATABASE rental CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.2 启动后端

```bash
cd server
go mod tidy                        # 首次需要，安装 Go 依赖
$env:DB_PASSWORD="root"   # 设置 MySQL 密码（Windows PowerShell）
# 或 set DB_PASSWORD=your_password  # Windows CMD
# 或 export DB_PASSWORD=your_password  # Linux/Mac
go run main.go
```

**环境变量说明**：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| DB_HOST | 127.0.0.1 | MySQL 地址 |
| DB_PORT | 3306 | MySQL 端口 |
| DB_USER | root | MySQL 用户 |
| DB_PASSWORD | (空) | MySQL 密码 |
| DB_NAME | rental | 数据库名 |
| JWT_SECRET | rental-system-secret-key-2026 | JWT 签名密钥 |
| SERVER_PORT | 8080 | 后端监听端口 |
| UPLOAD_DIR | ./storage/media | 媒体文件存储路径 |

启动后自动完成：建表（AutoMigrate）、种子超级管理员。

### 2.3 启动前端

```bash
cd web
npm install        # 首次需要
npm run dev        # 开发模式，默认 :5173
```

前端通过 Vite 代理 `/api` → `http://localhost:8080`，本地开发无需额外配置。

### 2.4 生产部署

```bash
# 构建前端
cd web
npm run build       # 输出到 dist/

# 后端编译
cd server
go build -o rental-server.exe .
```

推荐使用 Nginx 反向代理：

```
前端静态文件  →  Nginx → / 路径
后端 API      →  Nginx → /api 反向代理到 :8080
媒体文件      →  Nginx → /api/media/*filepath → storage/media 目录
```

### 2.5 快速使用流程

1. 启动后端和前端
2. 浏览器打开 `http://localhost:5173` → 自动跳转到公共楼栋列表页
3. 访问 `http://localhost:5173/login`，用 `admin / admin123` 登录（超级管理员）
4. 登录后自动跳转到平台管理后台 `http://localhost:5173/admin/buildings`
5. 创建楼栋 → 系统自动生成一个楼栋管理员账号
6. 用楼栋管理员账号登录，进入楼栋管理后台管理房间/账单/分红

### 3.2 多租户数据隔离方案

**策略**：单数据库 + `building_id` 行级隔离

所有业务表均包含 `building_id` 字段，通过中间件 `BuildingScopeMiddleware` 自动从 JWT 中提取 `building_id` 并注入查询条件，确保每个楼栋的管理员只能操作自己楼栋的数据。

**涉及 `building_id` 的表**：
- `users` — 用户所属楼栋
- `rooms` — 房间所属楼栋
- `rental_contracts` — 合同所属楼栋
- `bills` — 账单所属楼栋
- `tasks` — 任务所属楼栋
- `shareholders` — 股东所属楼栋
- `dividends` — 分红所属楼栋
- `room_media` — 通过 rooms 间接关联 building

### 3.7 核心业务逻辑

#### 房间生命周期

```
vacant（未出租）
  │  点击"设为已出租" → 创建合同 + 租客 + 按天租金账单
  ▼
rented（已出租）
  │  合同到期前 30 天自动标记
  ▼
expiring（即将退租）
  │  到期后自动标记
  ▼
expired（已到期）
  │  创建待办任务 → 管理员处理退租押金
  ▼
vacant（再次循环）
```

#### 自动租金账单

- **定时任务**：每月 1 号 0 点遍历所有楼栋的 active 合同，按当月天数比例生成租金收入账单
- **新出租时**：按当月剩余天数比例计算租金，生成按天折算的租金账单
- **同月再次出租**：累加金额并在备注追加修改记录

#### 押金处理规则

- **收取押金**：押金记在合同上，**不创建收入账单**
- **退租退押金**：全额退还 → 无账单；部分退还 → 只创建扣留部分的押金收入账单（不创建押金退还支出账单）
